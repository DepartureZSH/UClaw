## Project Overview

**UClaw** is an Electron desktop application that provides a graphical interface for [OpenClaw](https://github.com/OpenClaw) AI agents. It bundles the OpenClaw Python runtime and manages its lifecycle, exposing a React UI for chat, scheduled tasks, channel integrations, and skill management.

**二开目标（已完成）**：将 UClaw 改造为可从 U 盘直接运行的便携桌面 AI 客户端，内置 [new-api](https://github.com/Calcium-Ion/new-api)（OpenAI 兼容聚合接口）提供商支持，开箱即用。

## Commands

```bash
# Install dependencies (also downloads bundled uv binary)
pnpm run init

# Start development server (hot-reload, launches Electron)
pnpm dev

# Type-check without emitting
pnpm typecheck

# Lint and auto-fix
pnpm lint

# Run unit tests
pnpm test

# Run a single test file
pnpm vitest run src/path/to/file.test.ts

# Run E2E tests (Playwright)
pnpm test:e2e

# Full production build (Vite + OpenClaw bundle + electron-builder)
pnpm build

# Build only the Vite/React part
pnpm build:vite

# Package installers
pnpm package:win          # Windows NSIS 安装包
pnpm package:mac          # macOS DMG
pnpm package:linux        # Linux AppImage/deb/rpm

# Package portable zips
pnpm package:win:portable
pnpm package:mac:portable
pnpm package:linux:portable

# Build all platforms + assemble USB directory
pnpm package:portable:all
```

The `pnpm dev` predev hook runs `scripts/generate-ext-bridge.mjs` and `scripts/prepare-preinstalled-skills-dev.mjs` automatically before starting.

## Architecture

### Dual-Process Model

```
┌─────────────────────────────────────────────────┐
│  Electron Main Process (electron/)               │
│  ┌──────────────┐  ┌───────────────────────┐    │
│  │  Host API    │  │  Gateway Manager      │    │
│  │  port 13210  │  │  (OpenClaw lifecycle) │    │
│  └──────┬───────┘  └──────────┬────────────┘    │
│         │ IPC (contextBridge)  │ WebSocket       │
└─────────┼──────────────────────┼─────────────────┘
          │                      │
┌─────────▼──────────┐  ┌────────▼────────────────┐
│  React Renderer    │  │  OpenClaw Gateway        │
│  src/              │  │  (~/.openclaw/)          │
│  port 5173 (dev)   │  │  port 18789 (default)   │
└────────────────────┘  └─────────────────────────┘
```

- **Electron Main** (`electron/main/index.ts`): App entry, window lifecycle, system tray, IPC handler registration.
- **Host API** (`electron/api/`): HTTP server on port 13210, handles `settings`, `providers`, `gateway`, `chat`, `cron`, `channels` routes.
- **Gateway Manager** (`electron/gateway/manager.ts`): Spawns and supervises the OpenClaw Python process; communicates via JSON-RPC over WebSocket.
- **React Renderer** (`src/`): All UI. Communicates with Main via `contextBridge` IPC (production) or HTTP/WebSocket fallback (dev).

### Frontend API Layer

`src/lib/api-client.ts` is the single transport abstraction. It routes calls through three transports in priority order: `ipc → ws → http`. The `UNIFIED_CHANNELS` set defines which IPC channels are available. Gateway-specific calls use `src/lib/gateway-client.ts` (WebSocket to port 18789).

### Provider System

Provider definitions live in `electron/shared/providers/` and are shared between main and renderer processes:

- `types.ts`: Core types (`ProviderType`, `ProviderAccount`, `ProviderProtocol`, `ProviderSecret`)
- `registry.ts`: Static provider metadata (base URLs, env vars, default models)
- `electron/utils/provider-registry.ts`: Backend layer, adds env-only providers
- `electron/services/providers/`: Runtime account CRUD and config management
- `electron/services/secrets/secret-store.ts`: OS keychain integration for API keys

Supported protocols: `openai-completions`, `openai-responses`, `anthropic-messages`.

**new-api 集成**：`new-api` 已作为内置 Provider 类型注册（`electron/shared/providers/registry.ts`）。UI 中其他 Provider 均已隐藏（`src/lib/providers.ts` 中 `hidden: true`），Settings → AI 提供商页面只展示 new-api。Provider 设置页支持动态获取模型列表（`GET /api/provider-accounts/:id/models`）。

### Config Sync Flow

User settings flow one-way from UClaw → OpenClaw config file:

1. `electron-store` (`~/.uclaw/settings.json`) stores app settings
2. `electron/gateway/config-sync.ts → prepareGatewayLaunchContext()` reads settings, provider accounts, and API keys
3. Syncs them to `~/.openclaw/openclaw.json` and injects API keys as environment variables before spawning the gateway process

### State Management (Frontend)

Zustand stores in `src/stores/`: `chat`, `providers`, `settings`, `gateway`, `channels`, `skills`, `agents`, `cron`. Each store calls the API client to hydrate from the main process.

### Extensions System

`uclaw-extensions.json` (generated by `scripts/generate-ext-bridge.mjs`) maps extension IDs to main/renderer entry points. Built-in extensions are in `electron/extensions/builtin/`.

## USB Portable Mode

### 触发机制

在可执行文件目录或其任意父级目录（最多 4 层）创建 `data/` 文件夹即触发便携模式，无需任何参数或环境变量。多平台 USB 布局下各平台可执行文件放在子目录，共用 USB 根目录的 `data/`。

```
UClaw-USB/
├── windows/
│   ├── UClaw.exe
│   ├── resources/
│   └── locales/
├── macos-arm64/
│   └── UClaw.app/
├── linux/
│   ├── uclaw
│   ├── resources/
│   └── locales/
└── data/              ← 所有平台共用（触发便携模式）
    ├── uclaw/         ← userData（settings.json 等）
    └── openclaw/      ← OpenClaw 配置
```

### 实现要点

- **`electron/main/index.ts`**：`detectPortableDataDir()` 向上最多走 4 层查找 `data/`，支持多平台 USB 布局（macOS .app 内部路径深 4 层）。调用 `app.setPath('userData', ...)` 并设置 `process.env.UCLAW_PORTABLE_ROOT`。必须在 `requestSingleInstanceLock()` 之前执行。

- **`electron/utils/paths.ts`**：`getOpenClawConfigDir()` 检查 `UCLAW_PORTABLE_ROOT`，便携时返回 `<portableRoot>/openclaw/`。`getUClawConfigDir()` 返回 `<portableRoot>/uclaw/`。

- **`electron/gateway/config-sync.ts`**：便携模式下向网关进程注入 `OPENCLAW_HOME=<portableRoot>`，让网关所有路径（config、workspace、skills）均解析到 `<portableRoot>/.openclaw/` 下，不读写本机 `~/.openclaw/`（openclaw 原生支持 `OPENCLAW_HOME` env var）。

- **`electron/utils/store.ts`**：`createDefaultSettings()` 检查 `UCLAW_PORTABLE_ROOT`，便携模式下默认 `autoCheckUpdate: false`、`telemetryEnabled: false`、`launchAtStartup: false`。

- **Setup 向导**（`src/pages/Setup/index.tsx`）：便携模式下显示简化 3 步向导（欢迎 → AI 配置 → 完成），跳过 Runtime 检测和安装步骤。AI 配置步骤直接创建 new-api Provider 账号并设为默认。

- **IPC 通道**：`app:isPortable`、`app:portableRoot` 已注册，渲染层通过 `invokeIpc('app:isPortable')` 感知便携模式。
