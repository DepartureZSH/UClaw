# v0.4.0 Milestone 2 Diagnostics and Repair Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add one-click support diagnostics and make startup/runtime repair pages use the same issue, action, and diagnostics vocabulary.

**Architecture:** Build a Main-owned diagnostics package service that collects storage, startup, gateway, channel, runtime, and log summaries, then redacts secrets before any renderer-visible output. Startup loading and runtime error repair pages keep their current UI but replace local ad-hoc copy text with the shared diagnostics package, while repair actions are recorded in Main so support can see what the user already tried.

**Tech Stack:** Electron Main IPC, Host API route handlers, React 19, TypeScript, Vitest, Playwright Electron E2E, existing startup snapshot and gateway diagnostics APIs.

---

- [ ] **Step 3: Implement `diagnostics-redaction.ts`**

Create `electron/utils/diagnostics-redaction.ts`:

```ts
const SECRET_FIELD_PATTERN = /^(api[-_ ]?key|apikey|authorization|cookie|set-cookie|token|access[-_ ]?token|refresh[-_ ]?token|company[-_ ]?key|client[-_ ]?secret|secret|password|credential|oauth)$/i;
const SECRET_TEXT_PATTERNS: RegExp[] = [
  /\bBearer\s+[A-Za-z0-9._~+/=-]{8,}/gi,
  /\bsk-[A-Za-z0-9._-]{8,}/g,
  /\buclaw[_-]company[_-][A-Za-z0-9._-]{8,}/gi,
  /\b(api[-_ ]?key|company[-_ ]?key|token|authorization|cookie)\s*[:=]\s*["']?[^"'\s,;]+/gi,
];

export function redactDiagnosticsText(input: string): string {
  return SECRET_TEXT_PATTERNS.reduce(
    (text, pattern) => text.replace(pattern, (match) => {
      const separator = match.includes('=') ? '=' : match.includes(':') ? ':' : ' ';
      const prefix = match.split(separator)[0] ?? '';
      return prefix ? `${prefix}${separator} [REDACTED]` : '[REDACTED]';
    }),
    input,
  );
}

export function redactDiagnosticsValue<T>(value: T): T {
  if (typeof value === 'string') {
    return redactDiagnosticsText(value) as T;
  }
  if (Array.isArray(value)) {
    return value.map((item) => redactDiagnosticsValue(item)) as T;
  }
  if (value && typeof value === 'object') {
    const result: Record<string, unknown> = {};
    for (const [key, child] of Object.entries(value as Record<string, unknown>)) {
      result[key] = SECRET_FIELD_PATTERN.test(key)
        ? '[REDACTED]'
        : redactDiagnosticsValue(child);
    }
    return result as T;
  }
  return value;
}
```

## Implementation Notes

- Renderer must not read log files or scrape local paths directly.
- Do not include raw chat transcripts in the diagnostics package for this milestone.
- Do not include full provider stores, auth profiles, cookies, OAuth files, or environment dumps.
- Keep `/api/diagnostics/gateway-snapshot` backward compatible for the Channels page.
- `copy-diagnostics` should always produce some text, even if package collection fails.
- Support package JSON must be redacted before returning from Main.
- Milestone 3 can reuse this package when reporting update failures.
