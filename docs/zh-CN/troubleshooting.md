# 如何排查 UClaw 桌面应用的常见问题

> 修复 UClaw 桌面版中的网关启动延迟、模型列表为空、网络搜索失败、渠道连接问题和应用启动错误。

本页面涵盖 UClaw 用户最常遇到的问题，从网关启动延迟、模型列表为空，到渠道连接失败和应用启动问题。针对你的具体症状查阅对应部分，如需更多详情，请在**设置 → 网关 → 日志**中查看日志。

## 网关问题

### 网关无法启动或启动很慢

首次启动时，UClaw 需要准备本地 OpenClaw 运行时环境，这会花费额外时间。扫描新文件的杀毒软件或速度较慢的 USB 驱动器可能会将此时间延长至 60 秒。在采取任何操作前，请等待启动屏幕完成。

如果 60 秒后网关仍然无法连接：

1. 打开**设置 → 网关 → 日志**，查找错误信息。
2. 临时禁用 UClaw 安装文件夹的实时杀毒扫描，然后重试。
3. 如果你从 USB 驱动器运行，请将应用或工作区移至更快的驱动器（USB 3.0+ 或 SSD）并重新启动。

### 活动监视器或任务管理器中出现多个 UClaw 进程

这是预期行为，不是错误。UClaw 基于 Electron 构建，Electron 会将应用分成独立的操作系统进程——一个主进程、一个或多个渲染进程以及工具进程。你可能在活动监视器（macOS）或任务管理器（Windows）中看到三到五个 UClaw 条目，它们都属于同一个运行中的应用。

### 端口 18789 冲突

应该只有一个进程在 `127.0.0.1:18789` 上监听。如果网关因端口冲突错误而无法启动，另一个进程可能已在使用该端口——包括前一次会话遗留的 UClaw 进程。

要查找正在使用该端口的进程：

**macOS / Linux：**
```bash
lsof -nP -iTCP:18789 -sTCP:LISTEN
```

**Windows（PowerShell）：**
```powershell
Get-NetTCPConnection -LocalPort 18789 -State Listen
```

如果结果显示是一个过时的 UClaw 进程，请使用托盘图标 → **退出 UClaw** 完全退出，然后重新启动应用。

## 模型和 AI 问题

### 模型列表为空

模型列表为空通常意味着 New API 一键配置尚未完成，或存在网络问题导致 UClaw 无法访问远程服务。

1. 检查你是否在设置向导或**模型**页面完成了 New API 配置。
2. 确认网络连接正常。
3. 前往**模型**页面并点击**获取模型列表**。
4. 如果列表仍为空，重新运行一键配置流程，然后再次获取。

### 网络搜索不工作

网络搜索与主 AI 模型是分开配置的。如果普通聊天正常但网络搜索失败：

1. 打开**模型**页面并滚动到**网络搜索**部分。它应显示**已配置**。
2. 如果未显示已配置，点击**重新检测**或再次运行一键配置。
3. 重启 UClaw 后再次尝试搜索请求。

请求网络搜索结果时，请明确表达——例如："搜索今天的头条新闻并附上来源。"这会提示 UClaw 使用搜索模型，而不是依赖 AI 的训练数据。

### API 密钥被拒绝

如果 API 密钥验证失败：

1. 重新仔细输入密钥——直接从提供商控制台复制，避免多余空格或截断。
2. 确认密钥在你的提供商账号中未过期或未被撤销。

对于**自定义**（OpenAI 兼容）提供商：如果网关的 `/models` 端点因非认证原因被阻止，UClaw 会在密钥验证期间自动回退到探测 `/chat/completions`。如果该探测也失败，请检查你的自定义网关 URL 是否正确且可访问。

## 渠道问题

### 保存后渠道未连接

保存新渠道配置后，网关可能需要重启才能接收到更改。UClaw 在需要时会显示提示——按照提示操作，或前往**渠道**页面手动点击**重启网关**。在测试渠道之前，请等待网关状态指示器显示已连接。

### 无效的账号 ID 错误

渠道账号 ID 必须遵循以下规则：

* 只能使用小写字母（a–z）、数字（0–9）、下划线（`_`）和连字符（`-`）
* 最多 64 个字符
* 必须以字母或数字开头（不能以下划线或连字符开头）

基于二维码的渠道（如微信和 WhatsApp）会自动管理账号 ID——你只需按照应用内的二维码流程操作即可。

### 微信二维码未出现

微信二维码流程需要启用腾讯微信渠道插件。如果二维码未出现：

1. 前往**渠道**页面，确认微信渠道条目存在且插件已启用。
2. 如果渠道未列出，请添加并启用腾讯微信插件。
3. 从渠道页面重启网关，然后重试。

## 启动和应用问题

### 应用在启动时重新进入设置向导

UClaw 在找不到其配置使用的工作区时会进入设置向导。常见原因：

* 首次在新电脑上启动
* 工作区文件夹被移动、重命名或删除
* USB 驱动器更改了盘符（Windows）或挂载路径（macOS/Linux）

要修复此问题，请按照向导重新选择正确的工作区文件夹。如果你使用的是 USB 驱动器，请确保在启动 UClaw 前已连接好驱动器。

### macOS 显示路径中包含"AppTranslocation"

如果应用路径包含 `/AppTranslocation/`，说明 macOS 已将 `.app` 包隔离，因为它是直接从下载位置打开的，而没有先移动。这会导致应用从随机的只读路径运行，从而阻止其正确保存设置。

要修复此问题：

1. 将 `UClaw.app` 文件移至**应用程序**文件夹（或下载目录之外的其他永久位置）。
2. 运行[便携 USB 文档](./advanced/portable-usb.md)中描述的 `xattr` 清理操作。
3. 从新位置重新启动应用。

### USB 驱动器上的自动更新非常慢

UClaw 的更新包包含大量小文件。向 USB 驱动器写入大量小文件本身就很慢，无论驱动器的标称速度如何。要改善更新性能：

* 使用额外评定为高写入吞吐量的 USB 3.0 或更快驱动器。
* 使用便携 SSD 代替闪存驱动器。
* 如果速度至关重要，在驱动器连接到快速端口时进行更新，然后再将其移至常用设备。

## 安全

### 保护你的 API 密钥和凭据

避免分享包含 API 密钥、令牌或其他凭据的设置页面截图或任何显示这些信息的页面。即使是部分可见的密钥有时也可能被重建或滥用。

如果你不小心分享了包含密钥的截图，请在提供商控制台撤销该密钥并生成新的密钥。

### 在便携模式下保护聊天历史

你的工作区文件夹包含完整的对话历史和配置。如果你在 USB 驱动器上以便携模式使用 UClaw：

* 在驱动器上启用磁盘加密，以防驱动器丢失或被盗时保护数据。
* 不要将驱动器无人看管地留在公共电脑上。
* 避免将不受信任的文件直接放入工作区文件夹。
