# UClaw 设置：外观、网关与更新控制

> 在设置页面配置 UClaw 的外观、语言、启动行为、网关端口和日志、更新设置以及版本信息。

设置页面是 UClaw 的中央控制面板。你可以在这里更改应用外观、界面语言、是否随系统启动、本地网关的行为方式，以及 UClaw 处理软件更新的方式。随时点击侧边栏中的设置来调整这些选项。

## 通用

通用部分控制外观、语言和启动行为。

### 主题

UClaw 支持三种主题选项：

* **浅色** — 始终使用浅色配色方案
* **深色** — 始终使用深色配色方案
* **系统** — 跟随操作系统当前的外观设置，在操作系统级别切换深色模式时自动切换

要更改主题，前往**设置 → 通用**，从主题下拉列表中选择你的偏好。

### 语言

语言设置控制 UClaw 界面使用的语言。设置向导在支持的情况下会预选你的系统语言，否则默认为英语。你可以随时切换语言，无需重启。

> **说明：** 更改语言会立即更新界面。你的对话历史和智能体配置不受影响。

### 开机启动

启用**开机启动**可以让 UClaw 在你登录电脑时自动启动。如果你依赖 UClaw 执行定时任务或需要全天候可用的渠道，这非常有用。

要启用它，前往**设置 → 通用**并打开**开机启动**开关。

## 网关

网关部分显示 UClaw 为你管理的本地 OpenClaw 网关进程状态，并提供其端口、日志和重启行为的控制。

| 字段     | 说明                                                                                               |
| -------- | -------------------------------------------------------------------------------------------------- |
| **状态** | 显示网关当前是否已连接。已连接状态意味着 UClaw 可以与 AI 运行时通信。                              |
| **端口** | 网关监听的本地端口。默认为 `18789`。一次只应有一个进程使用此端口。                                 |
| **日志** | 在你的电脑上打开应用日志文件夹。排查问题或准备错误报告时使用。                                     |

### 网关重启行为

某些更改——例如保存代理设置或更新渠道配置——会自动重启网关。发生这种情况时，UClaw 在后台重启网关进程。网关重新连接后，现有对话将恢复。

> **说明：** 如果网关看起来卡住了，前往**设置 → 网关**并使用重启选项强制执行干净的重启。

## 更新

UClaw 可以直接在应用内检查和安装新版本。

### 手动检查

点击**检查更新**可立即查询 GitHub Releases 是否有新版本。如果有新版本，UClaw 会通知你。

### 自动检查

启用**启动时自动检查更新**可让 UClaw 在每次启动时静默检查更新。在下载或安装任何内容之前，你仍会收到提示。

### 自动更新

启用**自动更新**可允许 UClaw 自动下载并安装更新包。UClaw 将应用更新并提示你重启。

> **警告：** 如果你从慢速 USB 驱动器运行 UClaw，自动更新可能比预期慢得多。更新过程会写入大量小文件，慢速 USB 写入速度可能会导致进度停滞。如果可能，请使用快速 USB 3.0 驱动器或本地 SSD。

## 常见问题

**为什么应用语言在重启后恢复了？**

UClaw 会持久保存你的语言偏好。如果语言恢复，请检查 UClaw 是否具有对其设置目录的写入权限。在便携/USB 安装中，验证数据目录是否在可写卷上。

**网关状态显示"断开连接"，即使重启后也如此。**

打开**设置 → 网关 → 日志**查找启动错误。常见原因包括另一个进程已在使用端口 18789，或安全软件阻止了网关进程。要检查该端口上监听的是什么进程，在 macOS/Linux 上运行 `lsof -nP -iTCP:18789 -sTCP:LISTEN`，或在 Windows 的 PowerShell 中运行 `Get-NetTCPConnection -LocalPort 18789 -State Listen`。

**我启用了自动更新但什么都没发生。**

自动更新在后台下载安装包。UClaw 会在下载完成后显示重启并应用更新的提示。如果下载似乎停滞，请检查你的网络连接，并验证更新文件是否被防火墙或代理阻止。
