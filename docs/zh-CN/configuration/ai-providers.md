# 在 UClaw 中配置 AI 提供商和 API 密钥

> 在 UClaw 中配置 OpenAI、Anthropic 和自定义 OpenAI 兼容提供商。API 密钥安全存储在操作系统原生密钥链中，从不以明文存储。

UClaw 通过统一的配置面板连接 AI 提供商，无需手动编辑任何配置文件即可添加或切换提供商。你的凭据始终受到保护——UClaw 绝不以明文或任何磁盘上的文件形式存储 API 密钥。

> **说明：** UClaw 将所有 API 密钥存储在你操作系统的原生密钥链中（macOS Keychain、Windows 凭据管理器或 Linux 上的系统密钥存储）。密钥绝不会以明文形式写入磁盘，也不会包含在任何导出的配置文件中。

## 支持的提供商

UClaw 支持三种提供商类型：

* **OpenAI** — 通过 API 密钥连接，或通过浏览器 OAuth 登录（Codex 订阅）。同一提供商条目支持两种认证方式。
* **Anthropic** — 通过 Anthropic API 密钥连接。
* **自定义** — 任何 OpenAI 兼容的网关或代理端点。适用于自托管模型、企业 AI 网关或第三方 API 封装器。

> **提示：** 对于大多数用户，**New API** 是推荐的起点。它是 UClaw 默认的云管理端点，无需部署自己的基础设施或手动编辑 `openclaw.json` 即可提供 AI 响应和集成网络搜索。设置向导在首次启动时会为你配置好。

## 添加提供商

**步骤 1：打开 AI 提供商设置**

前往**设置 → AI 提供商**。你将看到已配置的提供商列表。

**步骤 2：打开提供商编辑器**

点击现有条目旁的**编辑提供商**，或点击**添加提供商**创建新的提供商。选择提供商类型：**OpenAI**、**Anthropic** 或**自定义**。

**步骤 3：输入你的凭据**

将 API 密钥粘贴到密钥字段中。对于 OpenAI，如果你有 Codex 订阅，也可以点击**通过浏览器登录**进行 OAuth 认证。

**步骤 4：配置自定义端点（仅限自定义提供商）**

输入你的 OpenAI 兼容网关的基础 URL。如果端点对兼容性敏感，设置**自定义 User-Agent** 以正确标识请求。

UClaw 会在网关因非认证原因拒绝 `/models` 端点时自动回退到轻量级的 `/chat/completions` 或 `/responses` 探测，因此即使在受限网关上，模型发现也能继续工作。

**步骤 5：保存并获取模型**

点击**保存**。然后导航到**模型**页面，点击**获取模型列表**以加载新添加提供商的可用模型。

## 每个智能体的模型覆盖

单个智能体可以覆盖全局默认模型，而不影响其他智能体。要为特定智能体设置模型，打开该智能体的设置并选择你希望该智能体使用的提供商和模型。未设置覆盖的智能体将继续使用全局默认值。

## 编辑或删除提供商

随时返回**设置 → AI 提供商**即可更新密钥、更改端点或删除不再需要的提供商。删除提供商也会从系统密钥链中删除其存储的凭据。
