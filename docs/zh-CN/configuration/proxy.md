# 为 UClaw 网关和渠道配置代理服务器

> 设置代理服务器，使 UClaw 网关、Electron 网络层和 Telegram 等渠道能够在受限网络上访问互联网。

如果你在企业网络、校园网络或通过本地代理客户端路由流量的 VPN 上，则需要告知 UClaw 如何访问互联网。UClaw 的代理设置同时应用于三个层面：Electron 应用本身、OpenClaw 网关进程，以及任何需要出站连接的渠道（如 Telegram）。在一处配置好，UClaw 负责处理其余所有事情。

## 打开代理设置

前往**设置 → 网关 → 代理**。

---

## 配置代理服务器

**步骤 1：输入代理服务器地址**

在**代理服务器**字段中输入本地代理的地址。此值将成为所有出站请求的默认代理。

```
http://127.0.0.1:7890
```

> **说明：** 如果你输入没有协议前缀的 `主机:端口` 值（例如 `127.0.0.1:7890`），UClaw 将其视为 HTTP 代理。

**步骤 2：添加绕过规则（可选）**

在**绕过规则**字段中，列出应直接连接而不经过代理的主机。条目之间用分号、逗号或换行符分隔。

```
localhost;127.0.0.1
```

匹配绕过规则的主机将直接连接。对内部服务和回环地址使用此选项。

**步骤 3：保存设置**

点击**保存**。UClaw 立即执行以下操作：

1. 重新应用代理到 Electron 的网络层
2. 使用新的代理配置重启 OpenClaw 网关

无需手动重启。

### 推荐配置

```
代理服务器：http://127.0.0.1:7890
绕过规则：localhost;127.0.0.1
```

---

## 代理设置与 Telegram 的交互

当 Telegram 作为渠道启用时，UClaw 会在保存时自动将代理设置同步到 OpenClaw 的 Telegram 渠道配置中。

> **说明：** 如果你在 UClaw 代理**禁用**时保存代理设置，UClaw 会从 OpenClaw 配置中明确清除 Telegram 渠道代理。这是删除之前设置的 Telegram 代理的正确方式。

> **提示：** 当网关重启且 UClaw 代理当前已禁用时，UClaw 会保留任何已设置的现有 Telegram 渠道代理。这可以防止因其他原因重启网关时意外重置。

---

## 回退行为

如果你将高级代理字段留空（见下方开发者模式部分），UClaw 会回退到**代理服务器**中的值处理所有请求类型。只有在需要为 HTTP、HTTPS 和 SOCKS 流量使用不同代理时，才需要填写高级字段。

---

## 开发者模式：按协议的代理覆盖

对于需要为不同协议使用不同代理的高级设置，请在**设置 → 高级 → 开发者**中启用开发者模式。这将显示三个额外字段：

| 字段                   | 用途                                        |
| ---------------------- | ------------------------------------------- |
| **HTTP 代理**          | 覆盖用于普通 HTTP 请求的代理                |
| **HTTPS 代理**         | 覆盖用于 HTTPS 请求的代理                   |
| **ALL_PROXY / SOCKS**  | 为遵守此变量的所有连接设置 SOCKS5 或全局代理|

> **提示：** 除非你有特定原因需要分割流量，否则请将这些字段留空。留空时，UClaw 对所有协议使用代理服务器的值。

---

## 常见问题

**我保存了代理设置但网关仍然无法连接。**

验证你的本地代理客户端（例如 Clash、V2Ray 或企业 VPN 客户端）是否正在运行并监听你输入的地址。然后检查**设置 → 网关**以确认网关已重启，且其状态显示已连接。

**如何完全禁用代理？**

清空**代理服务器**字段并保存。UClaw 将在没有任何代理的情况下重启网关。如果 Telegram 已启用，这也会从 OpenClaw 配置中清除 Telegram 渠道代理。

**更改代理设置后，我的 Telegram 渠道停止工作了。**

UClaw 在保存时将代理同步到 Telegram 的渠道配置。如果新代理不正确，请用正确的地址更新代理服务器字段并重新保存。如果你想专门删除 Telegram 代理，请禁用 UClaw 代理并保存——UClaw 将清除 Telegram 渠道代理。

**"视为 HTTP"对于裸 host:port 值意味着什么？**

如果你输入 `127.0.0.1:7890` 而非 `http://127.0.0.1:7890`，UClaw 会自动添加 `http://` 协议前缀。行为完全相同；显式协议形式只是更易读。
