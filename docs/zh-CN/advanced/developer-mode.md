# UClaw 桌面版开发者模式与启动诊断

> 启用开发者模式，访问 OpenClaw Doctor 诊断工具、高级代理覆盖、启动超时变量和网关排障工具。

开发者模式为需要深入了解应用运行状态的用户解锁了一套高级工具。包括 OpenClaw 网关的内置诊断命令、精细化的代理覆盖，以及允许你调整启动时序行为的环境变量。无需编辑任何配置文件即可访问这些功能。

## 启用开发者模式

前往**设置 → 高级 → 开发者**，打开**开发者模式**开关。启用后，本页面描述的高级控件将立即可见。

## OpenClaw Doctor

启用开发者模式后，**开发者**设置部分会显示两个诊断操作：

* **运行 Doctor** — 执行 `openclaw doctor --json` 并直接在 UI 中显示结构化诊断输出。使用此功能检查 OpenClaw 安装、网关进程和已连接工具的健康状况，无需打开终端。
* **运行 Doctor 修复** — 执行 `openclaw doctor --fix --yes --non-interactive`，自动修复 Doctor 识别的常见配置问题。

> **提示：** 先运行 Doctor 了解问题所在，再运行 Doctor 修复。JSON 输出准确显示哪些检查通过或失败以及原因。

## 高级代理覆盖

**设置 → 网关 → 代理**中的标准代理设置会将单个代理应用于 UClaw 所有出站流量。开发者模式添加了三个额外的覆盖字段，允许你独立路由不同类型的流量：

| 字段                   | 控制内容                                        |
| ---------------------- | ----------------------------------------------- |
| **HTTP 代理**          | 覆盖用于普通 HTTP 请求的代理                    |
| **HTTPS 代理**         | 覆盖用于 HTTPS 请求的代理                       |
| **ALL_PROXY / SOCKS**  | 为遵守此变量的所有连接设置 SOCKS 代理           |

将任何字段留空可回退到主**代理服务器**设置。

> **说明：** 保存任何代理设置——包括开发者模式覆盖——都会立即重新应用 Electron 网络层并自动重启网关进程。重启后，你的活跃对话将重新连接。

## 网关进程与单实例行为

UClaw 基于 Electron 构建，这意味着单个运行实例在操作系统的进程列表中通常会显示为多个进程（主进程、渲染进程、工具工作进程等）。这是正常行为，不是多个 UClaw 实例的迹象。

UClaw 使用 Electron 内置的应用锁加上本地进程文件锁（作为桌面会话总线不可用环境的备用方案）来强制执行单实例保护。在版本升级期间，新旧版本可能短暂共存，保护行为可能不对称。将所有运行实例升级到同一版本可解决此问题。

### 网关监听器

应该只有一个进程在监听网关端口。要验证：

**macOS / Linux：**
```bash
lsof -nP -iTCP:18789 -sTCP:LISTEN
```

**Windows（PowerShell）：**
```powershell
Get-NetTCPConnection -LocalPort 18789 -State Listen
```

如果你看到多个进程在监听 `127.0.0.1:18789`，请使用托盘菜单完全退出 UClaw（见下文），然后重新启动。

> **警告：** 如果两个进程都在监听端口 18789，UClaw 可能会出现不可预期的行为。关闭所有 UClaw 窗口，使用托盘菜单完全退出，等待几秒钟，然后重新打开应用程序。

## 托盘行为与完全关闭

点击窗口关闭按钮（**X**）会将 UClaw 隐藏到系统托盘——不会退出应用程序或停止网关进程。要完全关闭 UClaw，请右键点击托盘图标并选择**退出 UClaw**。

## 启动超时环境变量

对于排查启动缓慢或网关连接问题的高级用户，UClaw 提供了四个控制超时时间的环境变量。在启动 UClaw 之前在你的环境中设置这些变量：

| 变量                                         | 控制内容                                              |
| -------------------------------------------- | ----------------------------------------------------- |
| `UCLAW_STARTUP_TIMEOUT_GATEWAY_START_MS`     | UClaw 等待网关进程启动的最长时间                      |
| `UCLAW_GATEWAY_READY_WAIT_TIMEOUT_MS`        | UClaw 在启动后等待网关就绪的最长时间                  |
| `UCLAW_GATEWAY_CONNECT_HANDSHAKE_TIMEOUT_MS` | UClaw 与网关之间连接握手允许的最长时间                |
| `UCLAW_GATEWAY_CHALLENGE_TIMEOUT_MS`         | 网关质询-响应认证步骤允许的最长时间                   |

所有值的单位均为毫秒。这些变量用于在慢速硬件或网络受限环境中诊断启动失败——大多数用户不需要设置它们。
