# 完成 UClaw 首次启动设置向导

> UClaw 设置向导在首次启动时运行，引导你完成语言、工作区、环境检查、AI 配置和基础组件五个步骤。

当你首次打开 UClaw，或应用程序无法找到之前配置的工作区时，设置向导会自动启动。向导将引导你完成五个步骤以准备好运行环境。完成后，UClaw 将打开主聊天界面，你可以开始第一次对话。

> **提示：** 如果你已有来自其他安装的 OpenClaw 工作区，可以在向导中指向该文件夹。如果该文件夹已包含 `.openclaw/openclaw.json`，UClaw 将复用现有配置并跳过 AI 密钥输入步骤。

## 五个设置步骤

### 步骤 1：语言和地区

选择 UClaw 界面使用的语言。UClaw 会读取你操作系统的语言设置，并在支持的情况下预先选择匹配的语言。如果你的系统语言不在支持范围内，UClaw 默认使用英语。

从列表中选择你偏好的语言，然后点击**下一步**继续。

### 步骤 2：工作区选择

选择一个文件夹，UClaw 将在此存储你的 OpenClaw 数据——对话记录、智能体配置、技能状态和运行时文件。

点击**浏览**打开文件夹选择器，选择或创建你想使用的文件夹，然后点击**下一步**。

> **提示：** 如果你想在多台电脑上使用 UClaw，或从 USB 驱动器使用，请将工作区放在共享或便携驱动器上。任何运行 UClaw 的机器都可以打开同一个工作区文件夹，无需任何云账户即可保持对话和设置同步。

> **说明：** 工作区文件夹与 UClaw 本身的安装位置不同。你可以将工作区放在任何使用应用时可访问的地方——外置驱动器、网络共享或本地文件夹均可。

### 步骤 3：环境检查

UClaw 会验证内置 Python 运行时（由 `uv` 管理）和 OpenClaw 网关是否已就绪。此检查自动运行，通常只需几秒钟。

* **运行时**：UClaw 内置一个托管的 Python 3.12 环境。在 Windows 上，这已包含在官方发布包中，首次运行无需额外下载。
* **网关**：OpenClaw 网关是处理 AI 智能体执行的后台进程。UClaw 会自动启动它并等待其就绪。

如果任一检查失败，将显示错误信息和建议操作。大多数失败是由于安全软件阻止运行时或磁盘空间不足导致的。解决问题后点击**重试**。

> **警告：** 网关在首次启动时可能需要 10–30 秒才能就绪，在速度较慢的机器或 USB 驱动器上尤为如此。请等待状态指示器显示绿色已连接状态后再点击**下一步**。

### 步骤 4：AI 配置

配置智能体默认使用的 AI 提供商。输入你的 **New API 端点**（OpenAI 兼容 API 的基础 URL）和 **API 密钥**，然后从列表中选择默认模型。

点击**获取模型**从你的端点获取可用模型列表。选择想要使用的默认模型，然后点击**下一步**。

> **提示：** 如果你选择的工作区文件夹已包含具有有效提供商配置的 `.openclaw/openclaw.json`，此步骤将自动跳过。UClaw 将复用这些设置，而不是提示你重新输入。

> **说明：** UClaw 默认使用 New API 服务进行 AI 聊天和网络搜索。向导完成后，你可以随时在**模型**页面添加更多提供商或调整默认模型。

### 步骤 5：基础组件

UClaw 安装并启用一组内置技能以扩展智能体能力：

* **文档处理**：PDF、XLSX、DOCX 和 PPTX 的读取与摘要
* **网络搜索**：Brave Search 和 Tavily Search 集成
* **智能体工具**：`find-skills` 和 `self-improving-agent`

这些技能将部署到托管技能目录（默认为 `~/.openclaw/skills`）并自动启用。如果某个技能需要尚未配置的 API 密钥（例如 `BRAVE_SEARCH_API_KEY` 或 `TAVILY_API_KEY`），OpenClaw 将在该技能首次使用时提示配置。

点击**完成**以完成设置。

## 向导完成后

向导完成后，UClaw 将打开**聊天**页面。界面中的网关状态指示器显示 OpenClaw 运行时是否已连接。如果显示正在连接状态，请稍等片刻——网关可能仍在启动中。

- [与智能体对话](./features/chat.md) — 了解如何开始对话、使用网络搜索和指定特定智能体。
- [配置 AI 提供商](./configuration/ai-providers.md) — 添加或更新 API 凭据并更改默认模型。
- [连接消息渠道](./features/channels.md) — 将微信、Telegram、Discord 等平台与智能体关联。
- [管理技能](./features/skills.md) — 启用、禁用和配置智能体可用的技能。

## 重新运行向导

设置向导在以下情况下会再次自动运行：

* UClaw 首次在新电脑上启动
* 保存的工作区路径不再存在（例如，USB 驱动器挂载点已更改）
* 你删除了工作区文件夹

如需在不重新运行完整向导的情况下将 UClaw 指向不同的工作区，请前往**设置**并直接更新工作区路径。
