# 如何将消息渠道连接到 UClaw 中的 AI 智能体

> 将微信、Telegram、Discord、飞书、钉钉等平台与 AI 智能体关联，并在渠道页面管理账号和绑定关系。

渠道将外部消息平台连接到你的 AI 智能体，使微信、Telegram、Discord 等应用中的对话自动由你指定的智能体处理。每个渠道独立运行，且每个渠道可以有多个账号，每个账号绑定不同的智能体——这意味着你可以在单个 UClaw 安装中将不同账号的消息路由到不同的智能体。

## 添加渠道

**步骤 1：打开渠道页面**
点击左侧导航栏中的**渠道**。

**步骤 2：点击添加渠道**
点击**添加渠道**按钮，选择你要连接的平台。

**步骤 3：填写必填字段**
每种渠道类型有不同的必填字段。详见下方渠道参考表。

**步骤 4：设置账号 ID（如有提示）**
对于基于凭据的渠道，你可以输入自定义账号 ID。请参阅下方账号 ID 规则。

**步骤 5：保存并按提示操作**
点击**保存**。如果 UClaw 提示你重启网关，请按提示操作——渠道在网关重启完成前不会激活。

**步骤 6：绑定智能体**
渠道连接后，选择哪个智能体来处理该账号的传入消息。

> **提示：** 保存渠道后，UClaw 可能会显示重启网关的提示。渠道只有在重启完成后才会激活。

## 账号 ID 规则

手动输入自定义账号 ID 时，UClaw 会强制执行以下格式规则，以确保与路由系统的兼容性：

* 只能使用小写字母、数字、下划线（`_`）和连字符（`-`）
* 最多 64 个字符
* 必须以字母或数字开头（不能以下划线或连字符开头）

> **注意：** 不符合这些规则的账号 ID 将被拒绝。如果你连接的是基于二维码的渠道（如微信或 WhatsApp），账号 ID 会自动管理——无需手动输入。

## 每个渠道多个账号

单个渠道类型可以有多个账号。例如，你可以在同一渠道页面上连接两个 Telegram 机器人。对于每个账号，你可以：

* 设置**默认账号**——无特定目标时使用的账号
* 配置独立的**智能体绑定**——将每个账号的消息路由到不同的智能体

要切换默认账号，请点击渠道页面上的账号条目并切换默认设置。

## 渠道连接方式

UClaw 支持两种连接流程：

### 基于二维码的渠道

微信和 WhatsApp 使用二维码流程。UClaw 在应用内显示二维码，你用手机扫描即可。无需凭据或手动填写字段。

* **微信**：使用内置腾讯个人微信插件。点击微信渠道条目上的**连接**，用微信应用扫描二维码，并在手机上确认。
* **WhatsApp**：在手机上打开 WhatsApp，前往**已关联的设备**，扫描 UClaw 中显示的二维码。

> **提示：** 二维码会话可能在一段时间不活动后过期。如果渠道显示断开连接，请从渠道页面重新扫描二维码。

### 基于凭据的渠道

Telegram、Discord、飞书/Lark、钉钉、企业微信和 QQ 机器人使用从各平台开发者或管理门户获取的凭据。在 UClaw 中输入必填字段并保存。

详见下方完整渠道参考表中每个平台所需的字段。

## 渠道参考

| 渠道           | 连接方式   | 必填字段                                          |
| -------------- | ---------- | ------------------------------------------------- |
| **微信**       | 二维码     | 无需手动填写——用微信应用扫描应用内二维码          |
| **WhatsApp**   | 二维码     | 无需手动填写——通过 WhatsApp 已关联设备扫描        |
| **Telegram**   | 机器人令牌 | 机器人令牌、允许的用户 ID                         |
| **Discord**    | 机器人令牌 | 机器人令牌、服务器 ID、频道 ID（可选）            |
| **飞书 / Lark**| 应用凭据   | App ID、App Secret                                |
| **钉钉**       | 应用凭据   | Client ID / AppKey、Client Secret / AppSecret     |
| **企业微信**   | 应用凭据   | Bot ID、App Secret                                |
| **QQ 机器人**  | 应用凭据   | App ID、Client Secret                             |

### Telegram

从 Telegram 上的 [@BotFather](https://t.me/BotFather) 获取机器人令牌。**允许的用户 ID** 字段限制哪些 Telegram 用户可以与你的机器人交互——留空表示允许所有用户，或输入以逗号分隔的特定数字用户 ID。

### Discord

在 [Discord 开发者门户](https://discord.com/developers/applications)创建机器人并复制机器人令牌。**服务器 ID** 是你服务器的 ID（在 Discord 中右键点击服务器名称并选择**复制服务器 ID**）。可选的**频道 ID** 可将机器人限制在特定频道。

### 飞书 / Lark

在[飞书开放平台](https://open.feishu.cn/)创建企业自建应用，从应用凭据页面复制 **App ID** 和 **App Secret**。

### 钉钉

在钉钉开发者门户注册应用。UClaw 使用 Stream 模式，请从应用设置中输入 **Client ID**（也称 AppKey）和 **Client Secret**（也称 AppSecret）。

### 企业微信

从企业微信管理控制台或机器人配置页面获取 **Bot ID** 和 **App Secret**。

### QQ 机器人

在 [QQ 机器人开放平台](https://q.qq.com/)注册，复制 **App ID** 和 **Client Secret**。

## 诊断与重启网关

如果某个渠道显示异常状态，使用渠道页面上的**诊断**按钮检查连接详情。你可以复制诊断输出以在报告问题时分享。

如需直接从渠道页面重启网关，点击**重启网关**按钮。当你只需要快速重连时，这比导航到设置页面更快。

> **说明：** 网关必须运行，渠道才能接收和处理消息。如果网关状态指示器显示断开连接，请在排查各个渠道问题之前先重启网关。

## 故障排除

**渠道已保存但显示断开连接**
保存新渠道后，网关必须重启才能激活连接。如果 UClaw 提示你重启但你关闭了提示，请点击渠道页面上的**重启网关**手动触发重启。

**二维码在我扫描前过期**
微信和 WhatsApp 的二维码会话有有限的有效期。如果代码过期，关闭二维码对话框并重新打开渠道条目以生成新的二维码。

**账号 ID 字段显示验证错误**
账号 ID 只能使用小写字母、数字、下划线和连字符；必须以字母或数字开头；且不超过 64 个字符。检查是否有大写字母、空格或 `@`、`.` 等特殊字符并更正后再保存。

**消息已到达渠道但智能体没有回复**
检查渠道页面上该账号是否已设置智能体绑定。如果绑定为空，传入消息将不会路由到任何智能体。从绑定下拉列表中选择正确的智能体并保存。
