# 在 UClaw 中开始对话与管理聊天

> 在 UClaw 中开始对话、通过 @ 将消息路由到特定智能体、使用网络搜索、管理会话并查看网关状态。

聊天页面是 UClaw 的核心——你在这里通过现代对话界面与 AI 智能体交互。每次会话都保存在会话列表中，回复内容原生渲染 Markdown，实时网关状态指示器告诉你 AI 后端是否在线。每个智能体还可以独立配置模型覆盖，因此同一工作区中的不同智能体可以使用不同的 AI 提供商，互不影响。

## 开始新对话

点击聊天页面左上角的**新建聊天**，打开一个全新的会话。UClaw 每次都会创建新的对话上下文，与之前的会话保持隔离。

> **提示：** 如果你想继续之前的对话，请点击页面左侧会话列表中的对应条目，而不是新建聊天。

## 会话列表

左侧边栏的会话列表按最新时间顺序显示所有已保存的对话。点击任意会话即可切换并从上次停止的地方继续。UClaw 为每个会话存储完整的消息历史，内容不会丢失。

## 发送消息

在页面底部的输入框中输入消息，按 **Enter** 键或点击**发送**按钮。AI 生成回复期间，发送按钮会变为**停止**按钮——随时点击可中断回复生成。

如需附加文件，点击输入框工具栏中的附件图标。你可以在文字旁一并发送文档和其他文件。

> **提示：** 停止按钮只会暂停当前的生成。会话历史已保存到你停止的位置。

## 通过 @ 路由到特定智能体

如果你的工作区配置了多个智能体，可以在输入框中输入 `@智能体名称` 将消息直接发送给特定智能体。UClaw 会将消息直接路由到该智能体自己的对话上下文中——而非通过默认智能体——因此每个智能体的工作区保持独立。

**使用方法：**

1. 在输入框中输入 `@`，后接智能体名称。下拉列表将显示可用的智能体。
2. 从下拉列表中点击该智能体，或输入完整名称后按空格键确认。
3. 发送你的消息。消息将直接进入该智能体的上下文。同一会话中的后续消息将继续路由到该智能体。

> **说明：** 每个智能体维护各自独立的对话历史。切换回默认智能体时，会独立恢复其自己的上下文。

## 请求网络搜索

要让智能体搜索网络并在回答中包含来源，请在消息中明确提出。含糊的问题可能会从智能体的训练数据中回答，而非搜索网络。

触发网络搜索的示例提示词：

```
请搜索今天最新的科技新闻，并附上来源。
```

```
搜索 [软件名称] 的最新版本，总结有哪些变化。不要依赖你的训练数据。
```

```
查找 [服务] 的当前定价并附上来源链接。
```

> **提示：** 要求智能体"附上来源"或"不依赖训练数据"能得到更可靠的基于网络的回答。

## 每个智能体的模型覆盖

UClaw 中的每个智能体都可以使用与全局默认不同的 AI 提供商和模型。当智能体配置了模型覆盖时，它将专用该提供商。未设置覆盖的智能体将继承模型页面上设置的全局模型。

你可以在设置中的智能体配置内查看和编辑智能体级别的提供商覆盖。

## 网关状态指示器

聊天页面顶部附近有一个小状态指示器，显示 OpenClaw 网关是否已连接。如果显示警告或断开状态，你的消息将无法被处理，直到网关重新上线。

> **警告：** 如果网关显示断开连接，请前往**设置 → 网关**检查连接状态，或从**渠道**页面重启网关。

## 关闭 UClaw

点击**X**（关闭）按钮会将 UClaw 隐藏到系统托盘——**不会**退出应用程序或停止网关。如需完全退出 UClaw，请右键点击托盘图标并选择**退出 UClaw**。

> **提示：** 隐藏到托盘可以让网关在后台持续运行，使定时任务和渠道消息能够继续被处理。

## 常见问题

**我能同时打开多个对话吗？**
UClaw 将所有会话保存在会话列表中，你可以自由切换。但一次只有一个会话在输入框中处于活动状态。通过点击左侧列表中的会话来切换。

**我的对话会话存储在哪里？**
会话数据存储在你本地计算机上配置的工作区文件夹中。除 AI 提供商生成回复所需的内容外，不会将任何数据发送到外部服务器。

**为什么智能体在回复中途停止了？**
如果你点击了**停止**，是你主动取消了生成。如果智能体自行停止，网关可能已与提供商断开连接。请检查聊天页面顶部的网关状态指示器，如有需要，从**设置 → 网关**重启网关。

**@ 路由会在智能体间共享上下文吗？**
不会。当你使用 `@智能体名称` 时，UClaw 将消息路由到该智能体自己独立的对话上下文中。两个智能体不共享历史或记忆，除非显式配置为共享。
