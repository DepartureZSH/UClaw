# 如何在 UClaw 中配置 AI 模型和网络搜索

> 设置 AI 提供商、刷新可用模型列表、配置 Kimi/Moonshot 网络搜索，以及查看滚动 Token 使用历史。

模型页面是 AI 配置的核心控制面板。它显示当前活动的提供商、允许你获取最新可用模型列表，并展示支持聊天页面实时结果的网络搜索配置。你还可以查看 Token 使用的滚动历史记录，了解对话消耗了多少容量。

## AI 模型提供商配置

模型页面顶部的 **AI 模型提供商**部分显示：

* **提供商名称** — 提供 AI 能力的服务（大多数用户默认为 New API）
* **模型 ID** — 当前用于对话的已选模型
* **API 密钥状态** — 该提供商是否已保存有效密钥
* **获取模型列表**按钮 — 从提供商获取最新可用模型

**步骤：**

1. 点击左侧导航栏中的**模型**打开模型页面。
2. 确认 API 密钥状态显示为有效。如果显示错误，可能需要重新输入密钥或重新运行一键配置。
3. 点击**获取模型列表**从你的提供商拉取最新模型。模型选择器将自动更新。
4. 从更新后的列表中选择你想要的模型。你的选择将立即保存。

> **提示：** 大多数用户通过 New API 服务连接，该服务在设置向导期间已预配置。你无需部署自己的端点或手动编辑配置文件。

## 网络搜索配置

**网络搜索**部分出现在 AI 模型提供商部分下方。UClaw 通过 New API 使用 Kimi/Moonshot 风格的网络搜索，为智能体提供实时信息访问能力。

此部分显示：

* 网络搜索当前是否已配置
* 检测到的搜索模型（来自你 New API 账号的 Kimi/Moonshot 模型）
* 用于重新检查搜索设置的**检测**或**重新检测**按钮

**步骤：**

1. 查看网络搜索部分。它应显示搜索模型名称和"已配置"指示器。
2. 如果显示未配置或为空，点击**检测搜索模型**。UClaw 会扫描你的可用模型并识别正确的 Kimi/Moonshot 模型。
3. 前往聊天页面测试：`请搜索今天的头条新闻并附上来源。`

> **提示：** 网络搜索仅在你的 New API 模型列表中有兼容的 Kimi/Moonshot 模型时才有效。如果检测失败，先点击**获取模型列表**，然后重新检测。

## Token 使用历史

模型页面的下方部分显示 Token 消耗的滚动图表。你可以在以下视图间切换：

* **最近 7 天** — 过去一周的每日细分
* **最近 30 天** — 过去一个月的更宏观视图

两个时间窗口都是相对于今天的滚动窗口，而非固定的日历周期。图表按模型分组显示使用情况，让你了解哪些模型消耗了最多的 Token。

> **说明：** Token 使用情况从存储在工作区中的本地会话记录中读取。不会向外部分析服务发送任何数据。

## 故障排除

**模型列表为空**

通常意味着获取请求未完成。请尝试以下步骤：
1. 确认你的网络连接正常。
2. 再次点击模型页面上的**获取模型列表**。
3. 如果列表仍为空，前往**设置 → AI 提供商**重新运行一键配置，确保 API 密钥和端点已正确保存。
4. 检查**设置 → 网关 → 日志**中是否有来自提供商的错误信息。

**网络搜索失败但普通聊天正常**

普通对话和网络搜索使用不同的模型。你可能有可用的聊天模型但未检测到搜索模型。
1. 打开模型页面上的**网络搜索**部分。
2. 点击**检测搜索模型**重新扫描。
3. 如果找不到搜索模型，点击**获取模型列表**刷新模型目录，然后再次检测。
4. 如果问题持续，重启 UClaw 并重复检测步骤。

**API 密钥状态显示错误**

无效或过期的 API 密钥会阻止模型获取和聊天。
1. 前往**设置 → AI 提供商**并点击**编辑提供商**。
2. 重新输入你的 API 密钥并保存。
3. 返回模型页面并点击**获取模型列表**确认密钥有效。
