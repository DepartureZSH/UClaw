# 在 UClaw 中使用定时任务自动执行 AI 任务

> 创建定时提示词，让 AI 智能体自动按计划运行，无需手动操作即可推送报告、摘要和提醒。

定时任务是 UClaw 按照循环计划发送给 AI 智能体的预设提示词。你无需手动请求每日新闻摘要或每周报告，只需一次性定义任务，UClaw 就会自动运行——在你设定的时间、由你选择的智能体执行，并可选推送到外部消息渠道。无需手动编辑任何配置文件。

## 创建定时任务

**步骤 1：打开定时任务页面**
点击左侧导航栏中的**定时任务**。

**步骤 2：点击新建任务**
点击**新建任务**（或 **+**）按钮打开任务创建表单。

**步骤 3：输入任务名称**
为任务起一个简短、描述性的名称，以便后续在列表中识别（例如：`每日科技新闻`）。

**步骤 4：设置触发时间**
输入 cron 表达式或时间间隔来控制任务运行时间。请参阅下方的计划格式参考。

**步骤 5：编写提示词**
输入智能体在每次计划运行时将收到的提示词。提示词越具体，输出结果越稳定。

**步骤 6：选择执行智能体**
选择哪个智能体来处理此任务。请使用具备合适工具的智能体（例如，新闻任务应使用已启用网络搜索的智能体）。

**步骤 7：配置外部推送（可选）**
如果你希望将结果推送到消息渠道，请选择**发送账号**和**接收目标**。接收目标会从你连接的渠道中自动发现。

**步骤 8：启用并保存**
将任务切换为**已启用**并点击**保存**。任务将在下次计划时间运行。

## 计划格式

UClaw 支持两种定义任务运行时间的方式：

### Cron 表达式

使用标准五字段 cron 语法：`分钟 小时 日期 月份 星期`

| 表达式          | 含义                        |
| --------------- | --------------------------- |
| `0 8 * * *`     | 每天早上 8:00               |
| `0 9 * * 1`     | 每周一早上 9:00             |
| `30 18 * * 1-5` | 工作日晚上 6:30             |
| `0 */6 * * *`   | 每 6 小时                   |
| `0 8 1 * *`     | 每月 1 日早上 8:00          |

> **提示：** 当你需要精确控制计划时使用 cron 表达式，例如"仅限工作日"或"每月 1 日的特定时间"。

### 时间间隔

输入简单的间隔，如 `1h`、`30m` 或 `2h30m`，以该频率重复运行任务。

| 间隔   | 含义        |
| ------ | ----------- |
| `30m`  | 每 30 分钟  |
| `1h`   | 每小时      |
| `6h`   | 每 6 小时   |
| `24h`  | 每 24 小时  |

> **说明：** 间隔从任务保存或重新启用时开始计时，而非从午夜开始。

## 外部推送

如需将任务输出发送到消息渠道（如 Telegram 或微信），请在任务表单中配置推送字段：

* **发送账号** — 发送消息的渠道账号
* **接收目标** — 接收消息的用户、群组或对话

UClaw 会自动从你连接的渠道目录和会话历史中发现可用的接收目标。你无需手动查找 ID 或编辑任何配置文件——直接从下拉列表中选择即可。

> **说明：** 外部推送需要在渠道页面上至少连接并激活一个渠道。请在创建推送任务前先设置好渠道。

## 启用和禁用任务

列表中的每个任务都有一个**已启用**开关。关闭它可以暂停任务而不删除——当你想临时停止定时任务并稍后恢复时非常有用。已禁用的任务会以暗淡状态列出，便于识别。

## 示例任务

以下示例为常见用例提供参考：

### 每日新闻摘要

**任务名称：** 早间科技简报

**计划：** `0 8 * * 1-5`（工作日早上 8 点）

**提示词：**
```
搜索今天最重要的科技新闻。用要点总结 5 条最重要的新闻，并为每条附上来源链接。
```

### 定期网络搜索

**任务名称：** AI 研究摘要

**计划：** `0 12 * * *`（每天中午）

**提示词：**
```
搜索过去 24 小时内发表的关于大型语言模型的最新研究论文或公告。列出标题、每篇的一句话摘要以及来源链接。
```

### 定时项目状态报告

**任务名称：** 每周项目检查

**计划：** `0 9 * * 1`（每周一早上 9 点）

**提示词：**
```
总结近期对话中的所有待办事项，标出任何逾期项，并为本周提出三个优先事项建议。
```

### 推送到消息渠道的提醒

**任务名称：** 下班提醒

**计划：** `0 17 * * 1-5`（工作日下午 5 点）

**提示词：**
```
发送一条简短的下班提醒："该收工了。回顾你的待办事项，更新笔记，并为明天规划三个最优先的任务。"
```

启用**外部推送**，选择你的 Telegram 或微信发送账号，并选择接收目标，即可将此消息直接推送到你的手机。

> **警告：** 需要网络搜索的任务需要配置了网络搜索的智能体。如果智能体无法搜索网络，输出将依赖模型的训练数据而非当前信息。
